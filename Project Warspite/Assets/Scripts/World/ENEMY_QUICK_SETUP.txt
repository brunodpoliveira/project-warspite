  ENEMY PREFAB - QUICK SETUP CHECKLIST
═══════════════════════════════════════════════════════════════

STEP 1: CREATE BASE OBJECT
---------------------------
□ Create Capsule/Cylinder in scene
□ Rename: "Enemy_[Type]"
□ Position Y = 1 (sits on ground)
□ Scale appropriately (human-sized)


STEP 2: ADD COMPONENTS (in this order)
---------------------------------------
□ Add Component → Health (Warspite.Core)
    └─ Max Health: 100

□ Add Component → EnemyLogic (Warspite.World)
    └─ Config: [Drag EnemyConfig asset here]
    └─ Track Target: ✓ Checked
    └─ Min Spawn Distance: 1.5


STEP 3: CREATE MUZZLE POINT
----------------------------
□ Right-click enemy → Create Empty
□ Rename: "MuzzlePoint"
□ Position: X=0, Y=1.5, Z=0.5 (front of enemy)
□ Drag into "Muzzle Point" field in EnemyLogic


STEP 4: OPTIONAL - ADD PHYSICS
-------------------------------
□ Add Component → Rigidbody
    └─ Mass: 70
    └─ Use Gravity: ✓
    └─ Constraints: Freeze Rotation X, Y, Z


STEP 5: SAVE AS PREFAB
-----------------------
□ Drag enemy from Hierarchy → Project window
□ Save to: Assets/Prefabs/Enemies/
□ Name: "Enemy_[Type]"


STEP 6: TEST
------------
□ Drag prefab into scene
□ Press Play
□ Enemy should auto-target and fire at player

═══════════════════════════════════════════════════════════════
  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

❌ Enemy doesn't fire
   → Check Config is assigned in EnemyLogic
   → Check player has "Player" tag

❌ Bullets spawn inside enemy
   → Increase Min Spawn Distance to 2.0
   → Create MuzzlePoint child and assign it

❌ Enemy takes no damage
   → Check Health component is attached
   → Check Collider is NOT set to Trigger

❌ Console errors
   → Check all 6 configs were created (Warspite menu)
   → Check EnemyLogic.cs and EnemyConfig.cs compiled
